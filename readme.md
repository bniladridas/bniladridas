From this position (commit [a8ad049](https://github.com/harpertoken/harper/commit/a8ad049ffb51b806d1331f2581bf792914ebcd4a), [v0.1.3](https://github.com/harpertoken/harper/releases/tag/v0.1.3)), Harper is in early development and not fully production ready.
Reasons:

• Version: v0.1.3 indicates beta/alpha stage.
• Disabled features: MCP protocol is disabled due to dependency conflicts.
• Known issues: Recent [fixes](https://github.com/harpertoken/harper/pull/6) for CodeQL conflicts suggest ongoing stability work.
• Security: Has scans but may have vulnerabilities in early versions.
• Testing: Has unit tests, but limited integration testing.

It's functional for basic AI chat with supported providers, but for production use, wait for a stable v1.0+
release with full feature support and thorough testing. Check the GitHub [repo](https://github.com/harpertoken/harper) for updates.
